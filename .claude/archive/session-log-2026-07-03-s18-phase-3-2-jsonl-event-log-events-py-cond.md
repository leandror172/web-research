### Decisions Made

- Event sink = Protocol injection (`events` param), not log-handler capture or more callbacks ‚Äî matches repo's pluggable-component idiom; one `emit(dict)` absorbs future event types
- Stdlib JSON over structlog/eliot/python-json-logger ‚Äî schema ownership matters because the Auditor will later read these events
- `abandoned` vs `error` kept distinct ‚Äî replay analysis must exclude crashed sessions like `audit_failed` ones; can't recover the distinction later
- `queue_exhausted` vs `max_iterations` kept distinct ‚Äî same outcome, opposite tuning diagnosis (Auditor out of ideas vs budget too small)
- Session scoping: CLI per process, MCP per tool call (long-lived server would smear sessions)
- Ollama codegen: events.py generated by my-python-q3c30 with tests+repo files as context ‚Äî verdict 1 (improved; only defect an inline `__import__` wart)
